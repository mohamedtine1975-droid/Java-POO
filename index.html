01

C'est quoi la POO ?

🧠 POO = ?

Programmation Orientée Objet. C'est une façon d'organiser le code en regroupant les données et les comportements dans des objets.

🏗️ Le principe

On modélise le monde réel. Une voiture, un étudiant, un compte bancaire… tout peut devenir un objet dans notre programme.

🎯 Pourquoi ?

Code plus organisé, réutilisable, et plus facile à maintenir. C'est le standard dans l'industrie.

🔑 LES 4 PILIERS DE LA POO

Encapsulation Héritage Polymorphisme Abstraction

Pour le début, on va surtout voir Encapsulation et Héritage.

💡 ANALOGIE SIMPLE

Pense à un moule à gâteau (= la classe) et aux gâteaux qu'on produit (= les objets). Le moule définit la forme, mais chaque gâteau est unique.

02

La Classe

Une classe est un plan/modèle qui décrit ce que seront nos objets. Elle définit :

1

Les attributs

Les données / caractéristiques (ex: nom, âge, couleur)

2

Les méthodes

Les actions / comportements (ex: parler, calculer, afficher)

SYNTAXE DE BASE

public class NomClasse { }

✅ Commence par une majuscule
✅ Un fichier = une classe (même nom)
✅ Les accolades { } délimitent le contenu

Etudiant.java 
// Déclaration d'une classe "Etudiant"
public class Etudiant {

    // ── ATTRIBUTS (les données de l'objet) ──────────────
    String nom;
    int age;
    double moyenne;

    // ── MÉTHODES (les actions de l'objet) ───────────────
    void sePresenter() {
        System.out.println("Je m'appelle " + nom + ", j'ai " + age + " ans.");
    }

    void afficherMoyenne() {
        System.out.println("Ma moyenne est : " + moyenne);
    }

}
⚠️ À RETENIR

Une classe ne fait rien toute seule. C'est juste un plan. Pour l'utiliser, il faut créer un objet à partir de cette classe.

03

L'Objet (Instance)

Un objet est une instance d'une classe. C'est le "vrai" gâteau créé à partir du moule. On le crée avec le mot-clé new.

Main.java 
public class Main {
    public static void main(String[] args) {

        // Création d'un objet "etudiant1" de type Etudiant
        Etudiant etudiant1 = new Etudiant();

        // Accès aux attributs avec le point "."
        etudiant1.nom = "Yasmine";
        etudiant1.age = 20;
        etudiant1.moyenne = 14.5;

        // Appel de méthodes avec le point "."
        etudiant1.sePresenter();       // → "Je m'appelle Yasmine, j'ai 20 ans."
        etudiant1.afficherMoyenne();   // → "Ma moyenne est : 14.5"

        // On peut créer autant d'objets qu'on veut !
        Etudiant etudiant2 = new Etudiant();
        etudiant2.nom = "Karim";
        etudiant2.age = 22;
        etudiant2.sePresenter();       // → "Je m'appelle Karim, j'ai 22 ans."

    }
}
📌 SYNTAXE CRÉATION D'OBJET

TypeClasse nomObjet = new TypeClasse();

• TypeClasse = le type (nom de la classe)
• nomObjet = le nom de ta variable
• new = mot-clé qui crée l'objet en mémoire

🔗 ACCÈS AUX MEMBRES

On utilise le point ( . ) pour accéder aux attributs et méthodes d'un objet :

objet.attribut — lire/écrire une donnée
objet.methode() — exécuter une action

04

Le Constructeur

Le constructeur est une méthode spéciale qui s'exécute automatiquement à la création d'un objet. Il sert à initialiser les attributs dès le départ.

🔴 SANS constructeur (moins propre)

Sans constructeur 
Etudiant e = new Etudiant();
e.nom = "Ali";
e.age = 19;
e.moyenne = 13.0;
// 3 lignes juste pour remplir...

🟢 AVEC constructeur (propre et efficace)

Avec constructeur 
Etudiant e = new Etudiant("Ali", 19, 13.0);
// 1 seule ligne, direct !
Etudiant.java — avec constructeur 
public class Etudiant {

    // Attributs
    String nom;
    int age;
    double moyenne;

    // ── CONSTRUCTEUR ────────────────────────────────────
    // Même nom que la classe, PAS de type de retour !
    public Etudiant(String nom, int age, double moyenne) {
        this.nom = nom;         // "this" désigne l'objet en cours
        this.age = age;
        this.moyenne = moyenne;
    }

    // Méthodes
    void sePresenter() {
        System.out.println("Je m'appelle " + nom + ", j'ai " + age + " ans.");
    }

}

// Dans Main.java :
Etudiant e1 = new Etudiant("Sara", 21, 15.5);
Etudiant e2 = new Etudiant("Mehdi", 20, 11.0);
e1.sePresenter(); // "Je m'appelle Sara, j'ai 21 ans."
📛 Même nom que la classe

Si la classe s'appelle Etudiant, le constructeur s'appelle aussi Etudiant.

🚫 Pas de type de retour

Contrairement aux méthodes normales, le constructeur n'a pas de void ni de type de retour.

👆 Le mot-clé "this"

this.nom désigne l'attribut de l'objet courant, pour le distinguer du paramètre nom.

05

Attributs & Types

Les attributs sont les variables d'une classe. En Java, tout est typé — tu dois déclarer le type de chaque variable.

TYPE DESCRIPTION EXEMPLE VALEUR PAR DÉFAUT
int Nombre entier int age = 20; 0
double Nombre décimal double note = 14.5; 0.0
String Chaîne de caractères String nom = "Ali"; null
boolean Vrai ou Faux boolean actif = true; false
char Un seul caractère char genre = 'M'; '\u0000'
Voiture.java — exemple complet 
public class Voiture {

    // Attributs avec différents types
    String marque;        // String
    int annee;            // int
    double prix;          // double
    boolean estElectrique; // boolean

    // Constructeur
    public Voiture(String marque, int annee, double prix, boolean estElectrique) {
        this.marque = marque;
        this.annee = annee;
        this.prix = prix;
        this.estElectrique = estElectrique;
    }

    void afficherInfos() {
        System.out.println(marque + " (" + annee + ") - " + prix + "€");
        if (estElectrique) {
            System.out.println("⚡ Voiture électrique");
        }
    }

}

// Utilisation :
Voiture v1 = new Voiture("Tesla", 2023, 45000.0, true);
Voiture v2 = new Voiture("Renault", 2020, 15000.0, false);
v1.afficherInfos();
v2.afficherInfos();
06

Les Méthodes

Les méthodes définissent ce que peut faire un objet. Elles peuvent prendre des paramètres et retourner une valeur.

Structure d'une méthode 
// Structure générale :
// [modificateur] [typeRetour] nomMethode([paramètres]) { ... }

// Méthode sans retour (void) et sans paramètre
void direBonjour() {
    System.out.println("Bonjour !");
}

// Méthode sans retour AVEC paramètre
void saluer(String prenom) {
    System.out.println("Bonjour, " + prenom + " !");
}

// Méthode AVEC retour (int) et paramètre
int calculerAgeEnMois(int age) {
    return age * 12;   // "return" envoie la valeur
}

// Méthode AVEC retour booléen
boolean estMajeur(int age) {
    return age >= 18;
}

// Utilisation :
direBonjour();                        // → "Bonjour !"
saluer("Sara");                       // → "Bonjour, Sara !"
int mois = calculerAgeEnMois(20);     // → mois = 240
boolean majeur = estMajeur(17);       // → majeur = false
📚 TYPES DE RETOUR

void → ne retourne rien
int → retourne un entier
double → retourne un décimal
String → retourne du texte
boolean → retourne vrai/faux
NomClasse → retourne un objet !

⚠️ RÈGLE IMPORTANTE

Si le type de retour n'est pas void, tu DOIS utiliser return pour retourner une valeur, sinon Java donnera une erreur de compilation.

07

Encapsulation

L'encapsulation consiste à cacher les attributs et à ne les rendre accessibles qu'à travers des méthodes contrôlées (getters/setters).

🔐 LES MODIFICATEURS D'ACCÈS

public Accessible de partout
private Accessible seulement dans la classe
protected Accessible dans la classe et ses sous-classes

En POO, les attributs sont généralement private, les méthodes public.

CompteBancaire.java — encapsulation complète 
public class CompteBancaire {

    // Attributs PRIVÉS — personne ne peut y accéder directement
    private String proprietaire;
    private double solde;

    // Constructeur
    public CompteBancaire(String proprietaire, double soldeInitial) {
        this.proprietaire = proprietaire;
        this.solde = soldeInitial;
    }

    // ── GETTER — lire la valeur ──────────────────────────
    public String getProprietaire() {
        return proprietaire;
    }

    public double getSolde() {
        return solde;
    }

    // ── SETTER — modifier la valeur (avec contrôle !) ───
    public void setSolde(double montant) {
        if (montant >= 0) {          // Vérification avant modification
            this.solde = montant;
        } else {
            System.out.println("Erreur : montant invalide !");
        }
    }

    // Méthodes métier
    public void deposer(double montant) {
        if (montant > 0) {
            solde += montant;
            System.out.println("Dépôt de " + montant + "€ effectué.");
        }
    }

    public void retirer(double montant) {
        if (montant > 0 && montant <= solde) {
            solde -= montant;
            System.out.println("Retrait de " + montant + "€ effectué.");
        } else {
            System.out.println("Solde insuffisant !");
        }
    }

}

// Dans Main.java :
CompteBancaire compte = new CompteBancaire("Nadia", 1000.0);
// compte.solde = -5000;  ❌ ERREUR ! solde est private

compte.deposer(500);          // ✅ → "Dépôt de 500.0€ effectué."
compte.retirer(200);          // ✅ → "Retrait de 200.0€ effectué."
System.out.println(compte.getSolde()); // ✅ → 1300.0
MÉTHODE RÔLE NOMMAGE TYPE DE RETOUR
Getter Lire un attribut privé getNomAttribut() Type de l'attribut
Setter Modifier un attribut privé setNomAttribut(valeur) void
08

Héritage (Bonus)

L'héritage permet à une classe d'hériter des attributs et méthodes d'une autre classe. On évite de répéter du code !

🧬 CONCEPT

Classe parente (super-classe) → contient les éléments communs
Classe enfant (sous-classe) → hérite ET peut ajouter ses propres éléments

Mot-clé : extends

Héritage — Animal → Chien 
// ── CLASSE PARENTE ──────────────────────────────────────
public class Animal {
    protected String nom;
    protected int age;

    public Animal(String nom, int age) {
        this.nom = nom;
        this.age = age;
    }

    public void manger() {
        System.out.println(nom + " est en train de manger.");
    }

    public void dormir() {
        System.out.println(nom + " dort...");
    }
}

// ── CLASSE ENFANT ───────────────────────────────────────
// Chien HÉRITE de Animal → il a aussi nom, age, manger(), dormir()
public class Chien extends Animal {

    private String race; // Attribut spécifique au Chien

    // super() appelle le constructeur du parent
    public Chien(String nom, int age, String race) {
        super(nom, age);  // Appel du constructeur de Animal
        this.race = race;
    }

    // Méthode spécifique au Chien
    public void aboyer() {
        System.out.println(nom + " aboie : Wouf Wouf !");
    }

    // Override = redéfinir une méthode du parent
    @Override
    public void manger() {
        System.out.println(nom + " mange sa croquette avec enthousiasme !");
    }
}

// Dans Main.java :
Chien rex = new Chien("Rex", 3, "Berger Allemand");
rex.manger();   // → "Rex mange sa croquette avec enthousiasme !" (version Chien)
rex.dormir();   // → "Rex dort..." (hérité de Animal)
rex.aboyer();   // → "Rex aboie : Wouf Wouf !" (spécifique Chien)
🔼 super()

Appelle le constructeur ou une méthode de la classe parente. Toujours en première ligne du constructeur enfant.

🔄 @Override

Indique qu'on redéfinit une méthode héritée. L'annotation est optionnelle mais recommandée pour la clarté.

09

Classes Abstraites & Interfaces

🏗️ Classe abstraite

Une classe incomplète qu'on ne peut pas instancier directement. Elle définit une base commune et force les sous-classes à compléter certaines méthodes.

📋 Interface

Un contrat pur. Elle ne contient que des signatures de méthodes (ce qu'il faut faire) sans aucune implémentation (comment le faire).

🔑 DIFFÉRENCE CLÉS

abstract class peut avoir des attributs, des méthodes normales ET des méthodes abstraites
interface que des méthodes sans corps (par défaut) — c'est un contrat à 100%
extends pour hériter d'une classe abstraite  |  implements pour implémenter une interface

Partie 1 — La Classe Abstraite

Mot-clé : abstract. Une méthode abstraite n'a pas de corps — juste sa signature. Les sous-classes sont obligées de l'implémenter, sinon Java refuse de compiler.

Forme.java + Rectangle.java + Cercle.java 
// ── CLASSE ABSTRAITE ─────────────────────────────────────────
// "abstract" devant class = on NE PEUT PAS faire : new Forme()
// Elle sert uniquement de base pour les sous-classes.
public abstract class Forme {

    // Attribut normal (hérité par toutes les sous-classes)
    protected String couleur;

    // Constructeur normal
    public Forme(String couleur) {
        this.couleur = couleur;
    }

    // Méthode ABSTRAITE : pas de corps { }, juste la signature.
    // Chaque sous-classe DOIT obligatoirement la redéfinir.
    public abstract double calculerSurface();   // ← pas de { }
    public abstract double calculerPerimetre(); // ← pas de { }

    // Méthode NORMALE (héritée telle quelle, pas besoin de redéfinir)
    public void afficherCouleur() {
        System.out.println("Couleur : " + couleur);
    }
}

// ── SOUS-CLASSE 1 : Rectangle ────────────────────────────────
public class Rectangle extends Forme {

    private double largeur, hauteur;

    public Rectangle(String couleur, double largeur, double hauteur) {
        super(couleur);  // appel du constructeur de Forme
        this.largeur = largeur;
        this.hauteur = hauteur;
    }

    // Obligation de définir les méthodes abstraites de Forme
    @Override
    public double calculerSurface() {
        return largeur * hauteur;
    }

    @Override
    public double calculerPerimetre() {
        return 2 * (largeur + hauteur);
    }
}

// ── SOUS-CLASSE 2 : Cercle ───────────────────────────────────
public class Cercle extends Forme {

    private double rayon;

    public Cercle(String couleur, double rayon) {
        super(couleur);
        this.rayon = rayon;
    }

    @Override
    public double calculerSurface() {
        return Math.PI * rayon * rayon;  // Math.PI = 3.14159...
    }

    @Override
    public double calculerPerimetre() {
        return 2 * Math.PI * rayon;
    }
}

// ── UTILISATION dans Main.java ───────────────────────────────
// Forme f = new Forme("rouge");  ❌ ERREUR ! Forme est abstraite

Rectangle r = new Rectangle("bleu", 5.0, 3.0);
Cercle    c = new Cercle("rouge", 4.0);

System.out.println(r.calculerSurface());    // → 15.0
System.out.println(c.calculerSurface());    // → 50.26...
r.afficherCouleur();                        // → "Couleur : bleu" (hérité)
⚠️ RÈGLE IMPORTANTE

Si une sous-classe n'implémente pas toutes les méthodes abstraites, Java refusera de compiler — sauf si elle est elle-même déclarée abstract.

Partie 2 — L'Interface

Une interface est un contrat pur. Toutes ses méthodes sont implicitement public abstract. Une classe peut implémenter plusieurs interfaces (contrairement à l'héritage limité à une seule classe).

Payable.java (interface) + Commande.java 
// ── INTERFACE ────────────────────────────────────────────────
// Mot-clé "interface" (pas "class")
// Toutes les méthodes sont publiques et abstraites par défaut.
// Pas d'attributs d'instance, pas de constructeur.
public interface Payable {
    double calculerMontant();  // implicitement : public abstract
    void afficherFacture();    // implicitement : public abstract
}

// ── DEUXIÈME INTERFACE ───────────────────────────────────────
public interface Livrable {
    void livrer(String adresse);
}

// ── CLASSE QUI IMPLÉMENTE LES DEUX INTERFACES ────────────────
// "implements" + on peut mettre plusieurs interfaces séparées par ","
// La classe DOIT implémenter TOUTES les méthodes des deux interfaces.
public class Commande implements Payable, Livrable {

    private String produit;
    private int    quantite;
    private double prixUnitaire;

    public Commande(String produit, int quantite, double prixUnitaire) {
        this.produit      = produit;
        this.quantite     = quantite;
        this.prixUnitaire = prixUnitaire;
    }

    // ← Implémentation de Payable
    @Override
    public double calculerMontant() {
        return quantite * prixUnitaire;
    }

    @Override
    public void afficherFacture() {
        System.out.println("=== FACTURE ===");
        System.out.println("Produit  : " + produit);
        System.out.println("Quantité : " + quantite);
        System.out.printf ("Total    : %.2f DH%n", calculerMontant());
    }

    // ← Implémentation de Livrable
    @Override
    public void livrer(String adresse) {
        System.out.println("Livraison de " + produit + " → " + adresse);
    }
}

// ── UTILISATION ──────────────────────────────────────────────
Commande cmd = new Commande("Ordinateur", 2, 5500.0);
cmd.afficherFacture();       // → FACTURE avec total 11000 DH
cmd.livrer("Dakar, Sénégal"); // → Livraison de Ordinateur → Dakar

// Polymorphisme via interface : on peut stocker dans une variable de type interface
Payable p = new Commande("Clavier", 3, 150.0);
System.out.println(p.calculerMontant()); // → 450.0

Comparatif : Abstract vs Interface

CRITÈRE CLASSE ABSTRAITE INTERFACE
Mot-clé abstract class interface
Instanciable ? ❌ Non (new interdit) ❌ Non
Méthodes normales ? ✅ Oui ❌ Non (par défaut)
Attributs ? ✅ Oui ❌ Non (que des constantes)
Constructeur ? ✅ Oui ❌ Non
Héritage multiple ? ❌ Une seule (extends) ✅ Plusieurs (implements A, B)
Utilisé quand ? Relation "EST UN" avec code commun Relation "PEUT FAIRE" (contrat)
💡 ASTUCE POUR CHOISIR

→ Utilise une classe abstraite quand les sous-classes partagent du code commun (ex: toutes les Formes ont une couleur).
→ Utilise une interface quand tu veux juste garantir qu'une classe sait faire quelque chose, peu importe ce qu'elle est (ex: tout ce qui est Payable doit avoir calculerMontant()).

🎓 abstract class

Pense à un modèle incomplet. Animal est abstrait : on sait que tout animal fait un son, mais on ne peut pas définir lequel sans savoir l'espèce.

📝 interface

Pense à un CV de compétences. Si tu implémentes Nageur, tu t'engages à savoir nager — peu importe si tu es un dauphin ou un humain.

Cheat Sheet — À imprimer

Java POO — Résumé complet 
// ╔══════════════════════════════════════════════════════╗
// ║           JAVA POO — CHEAT SHEET COMPLET            ║
// ╚══════════════════════════════════════════════════════╝

// 1. DÉCLARATION DE CLASSE
public class MaClasse {
    // ...
}

// 2. ATTRIBUTS (variables de classe)
private String nom;          // private = encapsulé
private int age;
public double prix;          // public = accessible directement

// 3. CONSTRUCTEUR (même nom que la classe, pas de void)
public MaClasse(String nom, int age) {
    this.nom = nom;          // this = objet courant
    this.age = age;
}

// 4. GETTER (lire un attribut privé)
public String getNom() {
    return this.nom;
}

// 5. SETTER (modifier un attribut privé)
public void setNom(String nom) {
    this.nom = nom;
}

// 6. MÉTHODE
public void afficher() {          // void = pas de retour
    System.out.println(nom);
}

public int calculer(int x) {      // int = retourne un entier
    return x * 2;
}

// 7. CRÉER UN OBJET
MaClasse obj = new MaClasse("Test", 25);

// 8. UTILISER UN OBJET
obj.afficher();                   // appel méthode
String n = obj.getNom();          // appel getter
obj.setNom("Nouveau");            // appel setter

// 9. HÉRITAGE
public class SousClasse extends MaClasse {
    public SousClasse(String nom, int age) {
        super(nom, age);           // appel constructeur parent
    }

    @Override
    public void afficher() {       // redéfinition de méthode
        System.out.println("Override : " + nom);
    }
}

Exercice — Gestion de Produits (CRUD)

🗂️ 2 fichiers Java

Produit.java la classe objet, et GestionProduits.java le programme principal avec le menu.

🔄 CRUD complet

Ajouter, afficher tous, afficher par ID, modifier prix, modifier quantité, supprimer.

🧩 Concepts utilisés

Classe, constructeur, private, getters/setters, tableau d'objets, boucle do-while.

🎯 RÔLE DE CE FICHIER

C'est le plan de l'objet Produit. Il définit les données (attributs) et les actions (méthodes). Il ne contient pas de main() — il sert juste à créer des objets depuis l'autre fichier.

Produit.java 
// ─────────────────────────────────────────────────────────────
//  CLASSE PRODUIT — Le plan/modèle de notre objet
//  Un fichier .java = une classe publique (même nom que le fichier)
// ─────────────────────────────────────────────────────────────

public class Produit {

    // ── ATTRIBUTS ────────────────────────────────────────────
    // "private" = encapsulation.
    // Personne ne peut accéder à ces données directement depuis
    // l'extérieur. On DOIT passer par les getters/setters.
    private int    id;
    private String libelle;
    private int    quantite;
    private double prixUnitaire;
    private String datePeremption;   // format : "JJ/MM/AAAA"

    // ── CONSTRUCTEUR ─────────────────────────────────────────
    // Même nom que la classe, PAS de type de retour (pas de void).
    // S'exécute automatiquement quand on fait : new Produit(...)
    // "this.xxx" désigne l'attribut de CET objet pour le
    // distinguer du paramètre qui a le même nom.
    public Produit(int id, String libelle, int quantite,
                   double prixUnitaire, String datePeremption) {
        this.id             = id;
        this.libelle        = libelle;
        this.quantite       = quantite;
        this.prixUnitaire   = prixUnitaire;
        this.datePeremption = datePeremption;
    }

    // ── GETTERS — lire les attributs privés ──────────────────
    // "return" = renvoyer la valeur à celui qui appelle
    public int    getId()             { return id; }
    public String getLibelle()        { return libelle; }
    public int    getQuantite()       { return quantite; }
    public double getPrixUnitaire()   { return prixUnitaire; }
    public String getDatePeremption() { return datePeremption; }

    // ── SETTERS — modifier les attributs privés ──────────────
    // On ne crée un setter QUE pour les champs modifiables.
    // Ici : seulement prix et quantité (comme demandé dans le menu).
    public void setPrixUnitaire(double prixUnitaire) {
        this.prixUnitaire = prixUnitaire;
    }

    public void setQuantite(int quantite) {
        this.quantite = quantite;
    }

    // ── MÉTHODE D'AFFICHAGE ───────────────────────────────────
    // void = ne retourne rien, juste affiche dans la console.
    // System.out.printf permet de formater (%.2f = 2 décimales).
    public void afficher() {
        System.out.println("┌─────────────────────────────────────┐");
        System.out.println("  ID          : " + id);
        System.out.println("  Libellé     : " + libelle);
        System.out.println("  Quantité    : " + quantite);
        System.out.printf ("  Prix unit.  : %.2f DH%n", prixUnitaire);
        System.out.println("  Péremption  : " + datePeremption);
        System.out.println("└─────────────────────────────────────┘");
    }
}

▸ MÉTHODE CLÉ : trouverIndex()

⭐ LA MÉTHODE LA PLUS IMPORTANTE

Toutes les opérations sur un produit précis (afficher, modifier, supprimer) utilisent cette méthode. Elle retourne la position dans le tableau, ou -1 si l'ID n'existe pas.

GestionProduits.java — partie 2/4 
    // Cherche un produit par son id dans le tableau.
    // Retourne : l'index (position) si trouvé, ou -1 si pas trouvé.
    // Convention Java : retourner -1 = "non trouvé"
    static int trouverIndex(int id) {
        for (int i = 0; i < nbProduits; i++) {
            if (produits[i].getId() == id) {
                return i;   // trouvé ! on sort immédiatement
            }
        }
        return -1;   // on a tout parcouru, rien trouvé
    }

    // ── 1 — AJOUTER ──────────────────────────────────────────
    static void ajouterProduit() {
        if (nbProduits >= produits.length) {
            System.out.println("Tableau plein !"); return;
        }

        System.out.println("\n── Ajouter un produit ──");
        System.out.print("ID            : "); int id = sc.nextInt(); sc.nextLine();

        // Vérifier unicité de l'ID avant de continuer
        if (trouverIndex(id) != -1) {
            System.out.println("Cet ID existe déjà !"); return;
        }

        System.out.print("Libellé       : "); String lib  = sc.nextLine();
        System.out.print("Quantité      : "); int    qte  = sc.nextInt(); sc.nextLine();
        System.out.print("Prix unitaire : "); double prix = sc.nextDouble(); sc.nextLine();
        System.out.print("Date péremp.  : "); String date = sc.nextLine();

        // Créer l'objet et le mettre à la prochaine case libre
        produits[nbProduits] = new Produit(id, lib, qte, prix, date);
        nbProduits++;   // ← toujours incrémenter après ajout !
        System.out.println("✓ Produit ajouté !");
    }

    // ── 2 — AFFICHER TOUS ────────────────────────────────────
    // Important : boucler jusqu'à nbProduits, PAS produits.length
    // (sinon on lirait 97 cases null et on aurait une erreur !)
    static void afficherTous() {
        if (nbProduits == 0) { System.out.println("Aucun produit."); return; }
        System.out.println("\n── Liste (" + nbProduits + " produits) ──");
        for (int i = 0; i < nbProduits; i++) {
            produits[i].afficher();   // méthode de la classe Produit
        }
    }

    // ── 3 — AFFICHER PAR ID ──────────────────────────────────
    static void afficherParId() {
        System.out.print("\nID recherché : "); int id = sc.nextInt(); sc.nextLine();
        int index = trouverIndex(id);   // on cherche la position
        if (index == -1) { System.out.println("Introuvable !"); }
        else             { produits[index].afficher(); }
    }

GestionProduits.java — partie 4/4 
    // ── 4 — MODIFIER LE PRIX ─────────────────────────────────
    static void modifierPrix() {
        System.out.print("\nID du produit : "); int id = sc.nextInt(); sc.nextLine();
        int index = trouverIndex(id);
        if (index == -1) { System.out.println("Introuvable !"); return; }

        System.out.println("Prix actuel : " + produits[index].getPrixUnitaire());
        System.out.print("Nouveau prix : "); double prix = sc.nextDouble(); sc.nextLine();
        produits[index].setPrixUnitaire(prix);   // setter !
        System.out.println("✓ Prix modifié !");
    }

    // ── 5 — MODIFIER LA QUANTITÉ ─────────────────────────────
    static void modifierQuantite() {
        System.out.print("\nID du produit : "); int id = sc.nextInt(); sc.nextLine();
        int index = trouverIndex(id);
        if (index == -1) { System.out.println("Introuvable !"); return; }

        System.out.println("Quantité actuelle : " + produits[index].getQuantite());
        System.out.print("Nouvelle quantité : "); int qte = sc.nextInt(); sc.nextLine();
        produits[index].setQuantite(qte);   // setter !
        System.out.println("✓ Quantité modifiée !");
    }

    // ── 6 — SUPPRIMER ────────────────────────────────────────
    // Technique : décaler tous les éléments suivants vers la gauche
    //
    //  Avant : [Lait, Pain, Yaourt, Beurre]   supprimer Pain (index=1)
    //    i=1 : produits[1] = produits[2]  →  [Lait, Yaourt, Yaourt, Beurre]
    //    i=2 : produits[2] = produits[3]  →  [Lait, Yaourt, Beurre, Beurre]
    //  Après nbProduits-- : on ne "voit" plus le Beurre en double
    //  Résultat visible : [Lait, Yaourt, Beurre] ✓
    static void supprimerProduit() {
        System.out.print("\nID à supprimer : "); int id = sc.nextInt(); sc.nextLine();
        int index = trouverIndex(id);
        if (index == -1) { System.out.println("Introuvable !"); return; }

        for (int i = index; i < nbProduits - 1; i++) {
            produits[i] = produits[i + 1];   // décaler vers la gauche
        }
        produits[nbProduits - 1] = null;  // nettoyer la dernière case
        nbProduits--;
        System.out.println("✓ Produit supprimé !");
    }

}  // fin de la classe GestionProduits
⚠️ sc.nextLine() après nextInt()

Toujours vider le buffer après sc.nextInt() sinon le sc.nextLine() suivant lira une chaîne vide !

🔢 nbProduits vs length

Boucler jusqu'à nbProduits (cases utilisées), pas produits.length (100) sinon on lit des cases null.

🗑️ Suppression = décalage

Dans un tableau Java, on ne peut pas vraiment supprimer. On décale les éléments vers la gauche, puis nbProduits--.
